# 示例三：金融与投研知识库问答(RAG)
**中山大学 SUN YAT-SEN UNIVERSITY**

---

## 第17页内容
### 1. 数据准备与知识库构建:
- 数据收集: 获取一定量权威的垂直领域文档 (如上市公司的年度财报、招股说明书、深度行业研报或宏观经济分析报告)。
- 数据处理:
  - 格式转换: 将非结构化文档转换为纯文本格式。
  - 清洗降噪: 去除页眉页脚、页码、广告水印及乱码, 确保文本纯净度。

### 2. 大模型选型:
- 选择合适的大模型如GLM或者Qwen等。

### 3. 接入大模型:
- 云端API: 直接调用 DeepSeek API、阿里 DashScope 或硅基流动 (SiliconFlow)。优点: 无需配置高性能显卡, 开发门槛低, 适合快速验证投研 Demo。
- 本地私有化部署 (Local): 使用 Ollama 或 vLLM 框架进行本地推理; 优点: 数据不出域, 隐私性好, 适合高性能 PC。

---

## 第18页内容
### 3. RAG 工作流实现 :
- 技术框架: 使用低代码平台 (例如Dify, 基础得分要求) 或者代码实现 (加分项) 搭建 RAG 链路。
- 文本数据嵌入: 调用 Embedding 模型, 将清洗后的长文本切分为若干片段 (Chunks), 并转化为高维向量。
- 数据存储: 将生成的向量存入向量数据库, 推荐使用 FAISS 或者ChromaDB。
- 问题检索与生成: 系统接收用户提问, 在数据库中检索出 Top-N 个最相关的文本片段。将 "用户问题" 与 "相关片段" 拼接成 Prompt。提交给 LLM, 生成基于知识库的最终回答。

### 4. 用户交互界面:
- 界面设计: 简洁直观的聊天窗口。
- 交互体验: 支持流式输出与多轮对话。

## 第19页内容
考核要求: 基础得分75分, 需要完成基础的数据收集、处理与RAG构建, 能进行基本的检索问答。

| 类别 | 检查项 | 要求说明 |
| :--- | :--- | :--- |
| 实现形式 | 实现形式 | 低代码平台实现全流程，例如dify或者其他平台。 |
| 基础数据准备 | 知识库规模 | 收集至少 10份 金融相关文档（如研报、财报 PDF/TXT），内容需涵盖不同公司或业务。 |
| 基础数据准备 | 数据清洗与切分 | 实现文本提取与清洗（去除乱码/页眉等），并进行合理的文本切分。 |
| 核心流程构建 | 检索链路 | 实现 “输入问题 -> 检索Top-K相关片段” 的逻辑，并展示检索到的文本片段是否与问题相关。 |
| 核心流程构建 | 生成链路 | 接入大模型（API或本地模型），构建 Prompt 模板，将检索到的上下文填入 Prompt 并获取回答。 |
| 核心流程构建 | 系统产出 | 成功运行的脚本或服务，无需报错即可完成从 “提问” 到 “回答” 的全过程。 |
| 推理与演示 | 输入方式 | 支持用户输入具体的金融问题（如 “某公司去年的净利润是多少？”）。 |
| 推理与演示 | 测试范围 | 提问内容必须包含在知识库文档中，但不能是通用常识（验证模型是否真的查阅了文档）。 |
| 推理与演示 | 输出内容 | 输出结果需包含：1. 最终回答；2. 参考的文档来源或依据片段（这是RAG的核心特征）。 |

| :--- | :--- | :--- |
| 实现形式 | 代码实现 | 完全通过代码的方式实现所有功能。 |

| 知识整理 | Q&A 对的构建 | 机器切分的文档往往碎片化。针对研报中的关键数据（如 “某公司主要风险点”），提取数据并制作高质量的问答对或者知识点。 |

| 提交内容 | 课程报告 | 逻辑清晰，内容全面，描述准确，图文并茂，不能有过于明显的AI生成痕迹 |
